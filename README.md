# Android Dexing Bug
This repository aims to reproduce a bug with dexing a library with Android Gradle Plugin (AGP) 4.2.1.

This issue is still present in versions 7.0.0-beta04 and 7.1.0-alpha02.
This issue does not exist when using AGP 4.1.3.

### Testing
Try to build the app. The build fails during the `:app:desugarDebugFileDependencies` task.

To switch to AGP 4.1.3, update `gradle.properties` and do a clean build:
```properties
app.useNewAgp=false
```

### Overview
- The issue happens when trying to process a third-party library: `com.samsung:pass:1.2.2`.
